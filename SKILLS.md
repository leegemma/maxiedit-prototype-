# SKILLS

이 프로젝트에서 사용 가능한 Claude Code 스킬과 **언제 어떤 걸 쓰는지** 정리.
스킬 호출은 사용자 요청에 부합할 때만 — 사전 점검 없이 자동으로 돌리지 않는다.

## 자주 쓰는 것

### 🛠 `update-config`
**언제** — `.claude/settings.json` 또는 `settings.local.json` 의 변경이 필요할 때.
- "X 명령 권한 허용해줘" / "권한 추가" / "권한 옮겨줘"
- "환경 변수 X=Y 로 설정"
- "커밋 전에 X 실행되게 해줘" 같은 자동 동작 (PreCompact, PreToolUse 같은 hook)
- "before/after X" / "from now on when X" 패턴의 자동화

**언제 NOT** — Claude의 행동 규칙(피드백 메모리에 저장하면 되는 것)을 hook으로 강제하려는 경우. hook은 셸 명령만 실행 가능 → "요약" 같은 지능 요구 작업은 못함.

### 🧹 `simplify`
**언제** — 큰 기능 묶음 커밋 직후 또는 사용자가 명시적으로 "정리해줘" 했을 때.
- 변경된 코드의 재사용성·중복·과한 추상화 점검
- 리팩토링 안전성 자동 검증

**언제 NOT** — 작은 수정 커밋 (1~2줄). 디자인 변경 (CSS-only). 빠른 이터레이션 중.

### 🔍 `review`
**언제** — PR 검토 (`/review <PR번호>`) 또는 변경량이 큰 commit 후 셀프 리뷰.

**언제 NOT** — 매 커밋마다 자동 실행 (불필요한 비용).

### 🛡 `security-review`
**언제** — 외부 사용자에게 배포 직전, 큰 패치 전후, "보안 점검해줘" 요청 시.
- GitHub Pages 공개 후 / Capacitor 앱 배포 전 한 번 권장

**언제 NOT** — 단순 디자인/카피 변경.

### 🔕 `fewer-permission-prompts`
**언제** — 사용자가 동일한 bash 권한 팝업을 반복적으로 봐서 짜증날 때, 또는 "권한 자동화해줘" 요청 시.
- 트랜스크립트 스캔해서 read-only bash 패턴을 allowlist에 자동 추가
- 이미 한 번 적용했다 → `.claude/settings.json` 의 `permissions.allow` 확장됨

**언제 NOT** — 변경(write) 명령 권한 요청 (위험). 한 번만 쓰는 명령.

## 가끔 쓰는 것

### 📝 `init`
**언제** — 새 프로젝트의 첫 CLAUDE.md 만들 때 (이미 만든 상태라 거의 호출 안 함). 기존 파일 갱신은 init 대신 직접 편집.

### ⌨ `keybindings-help`
**언제** — 사용자가 Claude Code 자체의 단축키를 바꾸고 싶을 때 (이 프로젝트와 무관, 환경 설정).

### 🔁 `loop`
**언제** — 반복 작업 (예: "매 5분마다 빌드 상태 체크"). 이 프로젝트에선 거의 안 씀 — 빌드는 사용자 트리거.

### 📅 `schedule`
**언제** — 원격 cron 에이전트 운영. 이 프로젝트는 로컬/단발성 작업 위주라 미사용.

## 사실상 안 씀

### `claude-api`
이 프로젝트는 **Anthropic SDK / Claude API 를 쓰지 않음**. import한 적도 없고 앞으로도 계획 없음. 호출 금지.

## 호출 정책

- 사용자가 **명시적으로 슬래시 명령** (`/init`, `/review` 등) 으로 부르면 그대로 invoke.
- 사용자 요청에서 **트리거가 명확** (예: "권한 정리해줘" → fewer-permission-prompts) 하면 invoke 후 결과 보고.
- 트리거가 모호하면 **물어보지 말고 바로 안 씀** — 직접 작업으로 처리하는 게 더 빠름.
- 스킬 결과는 **요약해서 사용자에게 전달** (스킬 출력 자체를 그대로 토해내지 않음).
