# QA Checklist — 분기 회귀 테스트

분기마다 한 번씩 실기기로 돌리는 회귀 테스트 매뉴얼. 코드 안 건드려도 OS·브라우저는 알아서 바뀌니, 분기마다 같은 체크포인트를 다시 본다.

결과는 [HISTORY.md](../HISTORY.md)에 한 줄로 기록 (`YYYY-MM-DD QA 회귀: iOS XX OK / Android XX OK / APK OK` 형태). 회귀 발견 시 별도 커밋 또는 이슈로 분리.

## 테스트 결과 로그

| 회차 | 날짜 | 커밋 | iOS Safari | Android Chrome | Android APK | 메모 |
|---|---|---|---|---|---|---|
| (다음 회차) | YYYY-MM-DD | shorthash | ⏳ | ⏳ | ⏳ | |

표 위쪽이 최신. 각 칸은 ✅ / ❌ / ⚠️(부분 회귀) / `-`(미테스트).

---

## 사전 준비

1. `bash scripts/check-cachebuster.sh` 로 권장 `?v=N` 확인.
2. 위 N으로 라이브 URL 구성: `https://leegemma.github.io/maxiedit-prototype/?v=N`.
3. iOS Safari 최신 버전 + 한 단계 이전 버전 확인 (예: iOS 17 + 16).
4. Android Chrome 최신 버전.
5. Android APK는 `npm run cap:build:android` 으로 빌드 후 실기기 또는 Pixel 7 API 35 에뮬레이터에 설치.

## A. iOS Safari (최신 + 한 단계 이전)

### A-1. `app-frame`이 `100dvh` 추적
**왜**: iOS Safari URL bar가 스크롤에 따라 숨겨지면서 viewport 높이가 동적으로 바뀜. `100dvh`가 실제로 동적인지 확인.

**액션**: 홈 화면을 위로 살짝 스크롤해서 URL bar가 줄어드는 순간 `app-frame` 하단이 같이 늘어나는지 본다.

**합격**: 검정 배경이 화면 끝까지 차고, 흰 띠/잘림 없음.

**자주 깨지는 패턴**:
- iOS 베타 버전에서 `dvh`가 `vh`처럼 정적으로 동작 → 흰 띠 노출
- `meta name="viewport"`의 `viewport-fit=cover` 누락 시 노치 영역에서 black bar

### A-2. `navigator.share` 사진/영상 저장
**왜**: 결과 화면에서 다운로드 버튼이 시스템 share sheet를 띄우는데, iOS 정책 변경으로 `files` 옵션이 차단되면 즉시 폴백.

**액션**: 사진 1장 선택 → `다음` → 다운로드 → "사진으로 저장". MP4도 같은 동선.

**합격**: iOS 시스템 share sheet에 "사진 저장" 옵션 있고, 저장된 파일이 사진 앱에 떠야 함.

**자주 깨지는 패턴**:
- iOS가 `navigator.canShare({ files })` false 반환 → 앵커 다운로드로 폴백 (사진 앱에 자동 추가 안 됨)
- 영상이 너무 크면 share sheet 자체가 안 뜸 (1GB 한도 안에 있는지 확인)

### A-3. textedit 모달 — 키보드 올라올 때 안 가려짐
**왜**: iOS는 키보드 올라올 때 `visualViewport`가 줄어드는데 모달 위치가 그에 맞춰 안 움직이면 입력란이 키보드 뒤로 숨음.

**액션**: 결과 화면에서 텍스트 버튼 → 모달 떠서 input 클릭 → 키보드 올라옴.

**합격**: 입력 중인 행이 보이고, 확인 버튼이 노출됨.

**자주 깨지는 패턴**:
- 모달이 `position: fixed`로 굳어 있어서 키보드 올라와도 그대로 → 하단 가려짐

### A-4. 폰트 로드
**왜**: Google Fonts CDN 변경/차단/네트워크 이슈로 fallback 폰트로 떨어지면 디자인 의도 깨짐.

**액션**: 처음 진입 시 1~2초 내 Inter / Noto Sans KR / Nanum Myeongjo이 적용되는지.

**합격**: 본문은 Inter, 캡션은 명조체.

## B. Android Chrome (최신)

### B-1. MediaRecorder MP4 직접 출력 vs ffmpeg 폴백
**왜**: Android Chrome은 보통 MP4를 직접 뱉지만, 코덱 라이센스 이슈나 디바이스 차이로 폴백 타는 경우가 있음. ffmpeg.wasm은 첫 사용 시 ~25MB 다운로드라 사용자 체감 큼.

**액션**: 영상 1개 선택 → 결과 → MP4 저장. DevTools Network에서 ffmpeg-core wasm 다운로드 발생하는지 확인.

**합격(빠른 경로)**: ffmpeg-core 다운로드 없이 즉시 저장.
**합격(폴백)**: 다운로드 후 정상 저장. (다음 분기 점검 시 어떤 경로 탔는지 메모.)

### B-2. 영상 25분 / 1GB 한도
**왜**: 한도 안에서 OOM 없이 인코딩이 끝나는지.

**액션**: 24분~25분 짜리 영상 선택 후 저장. (한도 초과 시 토스트 출력 확인.)

**합격**: 25분 미만/1GB 미만이면 정상 저장. 초과 시 한국어 토스트로 거부.

### B-3. 텍스트 편집 한글 IME
**왜**: 한국어 입력기 동작이 브라우저별로 미묘하게 다름.

**액션**: 결과 → 텍스트 → 한글 입력 → 자모 결합 깨짐 없는지.

**합격**: 입력 그대로 반영. `autocomplete/autocorrect=off`로 자동 보정 안 됨.

## C. Android APK (실기기 또는 에뮬레이터)

### C-1. 시작 시 흰 플래시 없음
**왜**: Capacitor `backgroundColor: #000` 설정이 적용되면 launch부터 검정. 빠지면 흰 화면 한 프레임 노출.

**액션**: APK 콜드 스타트 (앱 종료 후 다시 열기) 5회.

**합격**: 5회 모두 검정 배경에서 시작.

### C-2. 미디어 권한 다이얼로그
**왜**: Android 14+ 부분 권한 (선택한 사진만)이 도입돼서 다이얼로그가 변경됨. 거부했을 때 앱이 멈추지 않는지.

**액션**: 첫 사진 선택 시 권한 다이얼로그 → "허용 안 함" 시도 → 다시 시도해서 허용.

**합격**: 거부했을 때 빈 picker 또는 안내 메시지. 허용 후 정상 picker.

### C-3. 시스템 뒤로가기
**왜**: Capacitor는 시스템 백 버튼을 페이지 네비게이션에 매핑. 잘못 매핑되면 홈으로 즉시 이탈.

**액션**: 홈 → 편집 → 결과 → 단일 화면. 각 단계에서 시스템 뒤로가기.

**합격**: 단일 → 결과 → 편집 → 홈 → (한 번 더) 종료 또는 그대로. 각 단계가 한 번씩.

### C-4. APK 크기
**왜**: 빌드별 크기 추이를 보면 의도치 않은 자산 증가가 보임.

**액션**: `android/app/build/outputs/apk/debug/app-debug.apk` 크기 확인.

**합격**: 분기마다 ±10% 이내. 갑자기 +30% 같은 점프면 추적.

## 회귀 발견 시

1. 어떤 항목이 깨졌는지 위 결과 표에 ❌ 또는 ⚠️로 기록.
2. **사소한 회귀**(스타일/문구)면 같은 회차 안에서 별도 커밋으로 수정 후 재테스트.
3. **큰 회귀**(MP4 저장 실패, 권한 모델 깨짐 등)이면:
   - 별도 GitHub 이슈 생성
   - 재현 환경 (OS 버전 + 브라우저 버전 + 디바이스 모델) 명시
   - 커밋과 별개로 추적
