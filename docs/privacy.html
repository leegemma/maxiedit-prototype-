개인정보처리방침 / Privacy Policy

MaxiEdit (com.leegemma.maxiedit)

Effective: 2026-04-30 · Last updated: 2026-05-13 (rev2)

요약 / Summary

사용자가 선택한 사진·영상은 디바이스 내부에서만 처리되며 어떤 서버에도 업로드되지 않습니다. 다만 무료 운영과 서비스 개선을 위해 (1) Google AdMob 광고 SDK가 광고 ID(IDFA/AAID)·IP·디바이스 정보를 수집·전송하고, (2) Google Firebase Analytics가 익명 사용 패턴(어떤 화면을 방문했고 어떤 기능을 썼는지 등의 집계 데이터)을 수집합니다. 개인을 식별할 수 있는 정보는 수집하지 않습니다.

Photos and videos you select are processed entirely on your device and are never uploaded to any server. However, to fund and improve the app, (1) Google AdMob collects and shares the advertising identifier (IDFA/AAID), IP, and basic device data, and (2) Google Firebase Analytics collects anonymous usage patterns (which screens are visited, which features are used, etc., as aggregate counts). No personally identifying data is collected.

수집·전송되는 데이터 / Data collected & shared

사진·영상 콘텐츠 / Your photos & videos

수집·전송되지 않습니다. 디바이스 메모리에서만 처리되고 앱 종료 시 사라집니다.

Not collected or transmitted. Processed in device memory only and discarded when the app exits.

광고 식별자 / Advertising identifier (IDFA / AAID)

Google AdMob이 광고 노출·빈도 캡·맞춤 광고를 위해 사용합니다. iOS 14.5+에서는 ATT(앱 추적 투명성) 권한 요청 후에만 추적 목적으로 사용되며, 사용자가 거부하면 비맞춤 광고만 노출됩니다.

Used by Google AdMob for ad delivery, frequency capping, and personalization. On iOS 14.5+, used for tracking only after the user grants App Tracking Transparency permission; if denied, only non-personalized ads are shown.

IP 주소·기기 정보 / IP address & device info

광고 요청 시 자동으로 AdMob 서버로 전송됩니다 (대략적 위치 추정·기기 호환성·사기 방지 목적). MaxiEdit 자체는 별도 저장하지 않습니다.

Automatically sent to AdMob servers with each ad request (for approximate geolocation, device compatibility, and fraud prevention). Not stored separately by MaxiEdit itself.

다운로드 횟수 / Download count

광고 게이트 동작을 위해 디바이스 내 `localStorage`에 누적 다운로드 횟수만 저장합니다. 외부로 전송되지 않으며, 앱 데이터 삭제 시 같이 삭제됩니다.

A cumulative download counter is stored in device-local `localStorage` so the ad gate can function. It is not transmitted anywhere and clears when app data is deleted.

분석 / Analytics (Google Firebase Analytics)

익명 사용 패턴(앱 실행, 화면 진입, 사진/영상 추가 횟수, 다운로드 시도/성공/실패, 광고 시청 완료 여부, 텍스트 정렬·뒤집기 등 기능 사용 빈도) 및 익명 디바이스 식별자(Google 광고 ID 또는 임시 인스턴스 ID), 대략적 위치(IP 기반 국가/도시 수준), OS·기기 모델·앱 버전을 Firebase로 전송합니다. 개별 사용자를 식별하거나 사진/영상 콘텐츠를 전송하지 않으며, 집계 형태로만 활용해 어떤 기능을 더 다듬어야 할지 판단합니다. Google 데이터 보존 기간은 기본 14개월입니다.

Anonymous usage patterns (app open, screen views, photo/video import counts, download attempts/success/failures, ad watch completion, feature usage such as text alignment and flip) along with an anonymous device identifier (advertising ID or transient instance ID), approximate location (IP-derived country/city), OS, device model, and app version are sent to Firebase. No individual user identification and no photo/video content are transmitted; only aggregate metrics are used to understand which features to improve. Google's default retention period is 14 months.

크래시 리포트 / Crash reports

현재 미사용. Firebase Crashlytics 도입 시 본 방침을 갱신합니다.

None currently. If Firebase Crashlytics is added later, this policy will be updated.

사용하는 권한 / Permissions used

사진·영상 라이브러리 접근 / Photo & video library access

편집할 사진/영상을 선택하기 위한 용도로만 사용됩니다. 선택한 미디어는 디바이스 메모리에서만 처리되며, 앱이 종료되면 사라집니다.

Used solely so you can pick the photos and videos you want to edit. Selected media stays in device memory only and is discarded when the app exits.

저장 공간 (선택) / Storage (optional)

완성한 콜라주를 사진/영상 파일로 디바이스에 저장하거나 시스템 공유 시트로 내보낼 때만 사용됩니다.

Used only when you save the finished collage as a photo or video file to your device, or share it via the system share sheet.

처리 위치 / Where processing happens

모든 편집·내보내기(이미지 합성, MP4 인코딩 등) 작업은 디바이스의 브라우저(WebView) 안에서 실행되며, 어떤 단계에서도 네트워크로 전송되지 않습니다.

All editing and export work (image composition, MP4 encoding, etc.) runs inside the device's browser/WebView. No step transmits anything over the network.

외부 자원 / Third-party resources

앱 UI를 표시하기 위해 다음 외부 자원을 처음 실행 시 다운로드합니다 (광고 외에는 사용자 식별 정보 포함하지 않음).

  • Google Fonts (Inter, Noto Sans KR, Nanum Myeongjo)
  • jsDelivr CDN — Sortable.js, html2canvas, ffmpeg.wasm (지연 로드)
  • Google AdMob — 광고 노출 및 IDFA/AAID·IP·기기 정보 수집·공유 (위 "수집·전송되는 데이터" 참고)
  • Google Firebase Analytics — 익명 사용 패턴·집계 데이터 수집 (위 "분석" 항목 참고)

Static assets above are fetched without identifiers; AdMob and Firebase Analytics fetches collect and share the data described in the "Data collected & shared" section.

전체 라이선스: licenses.html · AdMob 정책: Google Ads policy · Firebase 정책: Firebase privacy

아동 데이터 / Children's data

본 앱은 만 14세 이상 사용자를 대상으로 합니다. AdMob 광고 ID와 Firebase Analytics 익명 사용 데이터 수집이 만 14세 미만 아동을 표적으로 활용되지 않도록 두 SDK 모두 비아동 대상 모드로만 동작합니다(tagForChildDirectedTreatment 미적용 = 일반 사용자 모드).

This app targets users aged 14 and older. Both AdMob and Firebase Analytics operate in standard (non-child-directed) mode — tagForChildDirectedTreatment is not set — so no identifiers are tagged as targeting users under 14.

데이터 삭제 요청 / Data deletion request

MaxiEdit은 사용자를 개별적으로 식별할 수 있는 데이터를 자체적으로 저장하지 않습니다. 선택한 사진·영상은 디바이스 내부에서만 처리되며, 다운로드 카운터는 디바이스 로컬 `localStorage`에 저장되어 앱 데이터 삭제 시 함께 사라집니다.

광고 송출(Google AdMob)과 익명 사용 분석(Google Firebase Analytics)을 위해 수집되는 데이터는 Google이 보유하며, 다음 방법으로 직접 관리·삭제할 수 있습니다:

  • 광고 ID 재설정/삭제: Android — 설정 → Google → 광고 → 광고 ID 재설정 또는 삭제 / iOS — 설정 → 개인정보 보호 및 보안 → 추적 → MaxiEdit 권한 해제
  • Google 계정 활동 데이터 삭제: myactivity.google.com 에서 광고·분석 데이터 직접 관리

추가 문의나 데이터 처리 관련 요청은 다음 채널로 연락주세요. 영업일 기준 7일 이내에 응답드립니다.

  • 이메일: earlgreyggg@gmail.com
  • GitHub Issues: github.com/leegemma/maxiedit-prototype/issues

MaxiEdit does not directly store any personally identifiable data. Selected photos and videos are processed only on your device; the local download counter lives in `localStorage` and is wiped when app data is cleared. Data collected by Google AdMob and Firebase Analytics is held by Google and can be managed directly: reset/delete your advertising ID in OS settings, or visit myactivity.google.com. For additional data-related requests, contact earlgreyggg@gmail.com or open a GitHub issue — we respond within 7 business days.
